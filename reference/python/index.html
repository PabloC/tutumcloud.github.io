Python library¶

Overview¶

Tutum provides an open source Python library that wraps its HTTP REST API, making it easier to perform operations from Python applications.

Installing the Python library¶

In order to install the Tutum Python library, you can use pip install:

pip install python-tutum

It will install a Python module called tutum which you can use to interface with the API.

Authentication¶

You can use your ApiKey with the Python library in any of the following ways (will be used in this order):

  • Manually set it in your Python initialization code:
import tutum
tutum.user = "username"
tutum.apikey = "apikey"
  • Store it in a configuration file in ~/.tutum:
[auth]
user = "username"
apikey = "apikey"
  • Set the environment variables TUTUM_USER and TUTUM_APIKEY:
export TUTUM_USER=username
export TUTUM_APIKEY=apikey

To obtain your ApiKey, see Authentication.

Errors¶

The Python library will detect the API HTTP error status codes and raise the following exceptions with the error message, which should be handled by the calling application accordingly:

exception tutum.api.exceptions.TutumApiError¶

An error status code was returned when querying the HTTP API

exception tutum.api.exceptions.TutumAuthError¶

An 401 Unauthorized status code was returned when querying the API

Applications¶

class tutum.Application(**kwargs)¶

Represents a Tutum Application object

classmethod create(**kwargs)¶

Returns a new instance of the model (without saving it) with the attributes specified in kwargs

Returns:RESTModel – a new local instance of the model
delete()¶

Deletes the object in Tutum

Returns:bool – whether the operation was successful or not
classmethod fetch(pk)¶

Fetch an individual model given the pk

Parameters:pk (str) – The primary key for the object (usually UUID)
Returns:RESTModel – the instance fetched from Tutum
Raises:TutumApiError
classmethod list(**kwargs)¶

List all objects for the authenticated user, optionally filtered by kwargs

Returns:list – a list of objects that match the query
logs¶

Fetches and returns the logs for the application from Tutum

Returns:string – the current logs of the application
Raises:TutumApiError
refresh(force=False)¶

Reloads the object with remote information

Parameters:force (bool) – Force reloading even if there are pending changes
Returns:bool – whether the operation was successful or not
save()¶

Create or update the model in Tutum

Returns:bool – whether the operation was successful or not
start()¶

Starts the application in Tutum.

Returns:bool – whether or not the operation succeeded
Raises:TutumApiError
stop()¶

Stops the application in Tutum.

Returns:bool – whether or not the operation succeeded
Raises:TutumApiError

Example¶

>>> import tutum
>>> app = tutum.Application.create(image="tutum/hello-world", name="my-new-app", target_num_containers=2)
>>> app.save()
True
>>> app.state
"Starting"
>>> app.refresh()
>>> app.state
"Running"
>>> tutum.Application.list()
[<tutum.api.application.Application object at 0x10701ca90>]
>>> tutum.Application.list(name="my-new-app")[0].uuid
"fee900c6-97da-46b3-a21c-e2b50ed07015"
>>> app = tutum.Application.fetch("fee900c6-97da-46b3-a21c-e2b50ed07015")
>>> app.name
"my-new-app"
>>> app.target_num_containers = 3
>>> app.save()
True
>>> app.state
"Scaling"
>>> app.refresh()
>>> app.state
"Running"
>>> app.stop()
True
>>> app.start()
True
>>> app.delete()
True
>>> app.state
"Terminating"
>>> app.refresh()
>>> app.state
"Terminated"

Authentication utilities¶

tutum.api.auth.authenticate(username, password)¶

Authenticates a Tutum user using username and password. If successful, automatically sets tutum.user and tutum.apikey

Parameters:
  • username (str.) – The username of the user to authenticate
  • password (str.) – The password of the user to authenticate
Raises:

TutumAuthError
tutum.api.auth.get_auth(username, password)¶

Returns the user’s Username and ApiKey, or raises an exception if username/password incorrect

Parameters:
  • username (str) – The username/email of the user to authenticate
  • password (str) – The password of the user to authenticate
Raises:

TutumAuthError
Returns:

str, str – the Username, ApiKey to use for the given username/email
tutum.api.auth.is_authenticated()¶

Returns whether the tutum user and apikey are set

Returns:bool – whether the tutum user and apikey are set
tutum.api.auth.load_from_file(file='~/.tutum')¶

Attempts to read tutum’s credentials from a config file and return a tuple of (user, apikey)

Parameters:file (str) – The filename where Tutum auth config is stored
Returns:tuple – tuple of (user, apikey) if config found and valid, (None, None) otherwise
tutum.api.auth.logout()¶

Sets the tutum user and apikey to None
