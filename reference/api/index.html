Introduction¶

Tutum currently offers an HTTP REST API and a Python library that wraps this API. In this document you will find all the operations currently supported in the platform and examples on how to execute them as raw HTTP requests and by using the Python library.

Installing the Python library¶

In order to install the Tutum Python library, you can use pip install:

pip install python-tutum

It will install a Python module called tutum which you can use to interface with the API.

Authorization¶

In order to be able to make requests to the API, you should first obtain an ApiKey for your account. For this, log into Tutum, click on the menu on the upper right corner of the screen, and select Get Api Key

HTTP API¶

The Tutum HTTP API is reachable through the following hostname:

https://app.tutum.co/

All requests should be sent to this endpoint with the following Authorization header:

curl -H "Authorization: ApiKey username:apikey" https://app.tutum.co/api/v1/application/

Python library¶

You can use your ApiKey with the Python library in any of the following ways (will be used in this order):

  • Set the environment variables TUTUM_USER and TUTUM_APIKEY:
export TUTUM_USER=username
export TUTUM_APIKEY=apikey
  • Manually set it in your Python initialization code:
import tutum
tutum.user = "username"
tutum.apikey = "apikey"

Errors¶

Errors in the HTTP API will be returned with status codes in the 4xx and 5xx ranges.

The Python library will detect this status codes and raise TutumApiError exceptions with the error message, which should be handled by the calling application accordingly.

Host: app.tutum.co

Authorization: ApiKey username:apikey

Create and launch a new application¶

POST /api/v1/application/¶

Creates and deploys a new application

Example request:

Host: app.tutum.co
Accept: application/json

    "container_size": "XS"

    "stopped_num_containers": 0,

    "uuid": "80ff1635-2d56-478d-a97f-9b59c720e513",
    "web_public_dns": "my-new-app.alpha.tutum.io"
}
Json Parameters:
 
  • image_tag (string) – required, the resource URI of the image used to deploy this application, i.e. /api/v1/image/tutum/hello-world/tag/latest/
  • image (string) – optional, the image used to deploy this application in docker format, i.e. tutum/hello-world. Required if image_tag is not provided.
  • name (string) – optional, a human-readable name for the application, i.e. my-hello-world-app (default: image_tag without namespace)
  • container_size (string) – optional, the size of the application containers, i.e. M (default: XS, possible values: XS, S, M, L, XL)
  • target_num_containers (int) – the number of containers to run for this application (default: 1)
  • run_command (string) – optional, the command used to start the application containers, i.e. /run.sh (default: as defined in the image)
  • entrypoint (string) – optional, the command prefix used to start the application containers, i.e. /usr/sbin/sshd (default: as defined in the image)
  • container_ports (array(object)) – optional, an array of objects with port information to be exposed in the application containers, i.e. [{"protocol": "tcp", "inner_port": 80}] (default: as defined in the image)
  • container_envvars (array(object)) – optional, an array of objects with environment variables to be set in the application containers on launch, i.e. [{"key": "DB_PASSWORD", "value": "mypass"}] (default: as defined in the image, plus any link- or role-generated variables)
  • linked_to_application (array(object)) – optional, an array of application resource URIs to link this application to, i.e. ["/api/v1/application/80ff1635-2d56-478d-a97f-9b59c720e513/"] (default: empty array)
  • autorestart (string) – optional, whether the containers should be restarted if they stop, i.e. ALWAYS (default: OFF, possible values: OFF, ON_FAILURE, ALWAYS)
  • autoreplace (string) – optional, whether the containers should be replaced with a new one if they stop, i.e. ALWAYS (default: OFF, possible values: OFF, ON_FAILURE, ALWAYS)
  • autodestroy (string) – optional, whether the containers should be terminated if they stop, i.e. OFF (default: OFF, possible values: OFF, ON_FAILURE, ALWAYS)
  • roles (string) – optional, a list of Tutum API role resource URIs to grant the application, i.e. ["/api/v1/role/global/"] (default: empty array, options: see Roles)
Request Headers:
 
  • Content-Type – required, only application/json is supported
  • Authorization – required ApiKey authentication header in the format ApiKey username:apikey
  • Accept – required, only application/json is supported
Status Codes:
  • 202 – operation accepted
  • 400 – cannot perform the operation (probably the application is not in a suitable state)
  • 401 – unauthorized (wrong credentials)

Python library example

>>> import tutum
>>> app = tutum.Application.create(image="tutum/hello-world", name="my-new-app", target_num_containers=2, container_size="XS")
>>> app.save()
True

Terminate an application¶

DELETE /api/v1/application/(uuid)/¶

Roles¶

List all available roles¶

GET /api/v1/role/¶

This operation returns a list of all available roles to be used when launching an application.

Example request:

GET /api/v1/role/ HTTP/1.1
Host: app.tutum.co
Accept: application/json
Authorization: ApiKey username:apikey

Example response:

HTTP/1.1 200 OK
Cache-Control: must-revalidate, max-age=0
Content-Type: application/json
Vary: Accept, Authorization, Cookie

{
    "meta": {
        "limit": 25,
        "next": null,
        "offset": 0,
        "previous": null,
        "total_count": 1
    },
    "objects": [
        {
            "label": "Full access",
            "resource_uri": "/api/v1/role/global/",
            "scope": "global"
        }
    ]
}
Request Headers:
 
  • Authorization – required ApiKey authentication header in the format ApiKey username:apikey
  • Accept – required, only application/json is supported
Query Parameters:
 
  • offset (int) – optional, start the list skipping the first offset records (default: 0)
  • limit (int) – optional, only return at most limit records (default: 25, max: 100)
Status Codes:
  • 200 – no error
  • 401 – unauthorized (wrong credentials)

Python library example

>>> import tutum
>>> tutum.Role.list()
[<tutum.api.role.Role object at 0x10701ca90>]

Role objects have all the attributes of the returned JSON as properties
