Introduction¶

Tutum currently offers an HTTP REST API which is used by both the Web UI and the CLI. In this document you will find all the operations currently supported in the platform and examples on how to execute them as raw HTTP requests.

Authentication¶

In order to be able to make requests to the API, you should first obtain an ApiKey for your account. For this, log into Tutum, click on the menu on the upper right corner of the screen, and select Get Api Key

The Tutum HTTP API is reachable through the following hostname:

https://app.tutum.co/

All requests should be sent to this endpoint with the following Authorization header:

curl -H "Authorization: ApiKey username:apikey" https://app.tutum.co/api/v1/application/

List all applications¶

GET /api/v1/application/¶

This operation returns a list of all active and recently terminated (less than 5 minutes ago) applications.

Example request:

GET /api/v1/application/ HTTP/1.1
Host: app.tutum.co
Accept: application/json
Authorization: ApiKey username:apikey

Request Headers:
 
  • Authorization – required ApiKey authentication header in the format ApiKey username:apikey
  • Accept – required, only application/json is supported
Query Parameters:
 
  • offset (int) – optional, start the list skipping the first offset records (default: 0)
  • limit (int) – optional, only return at most limit records (default: 25, max: 100)
  • name (string) – optional, filter applications by name
  • uuid (string) – optional, filter applications by UUID
  • uuid__startswith (string) – optional, filter applications by UUIDs that start with the given string
  • state (string) – optional, filter applications by state
Status Codes:
  • 200 – no error
  • 401 – unauthorized (wrong credentials)

    "web_public_dns": "awesome-app.example.com"
}
Json Parameters:
 
  • image (string) – required, the image used to deploy this application in docker format, i.e. tutum/hello-world.
  • name (string) – optional, a human-readable name for the application, i.e. my-hello-world-app (default: image_tag without namespace)
  • container_size (string) – optional, the size of the application containers, i.e. M (default: XS, possible values: XS, S, M, L, XL)
  • target_num_containers (int) – the number of containers to run for this application (default: 1)
  • run_command (string) – optional, the command used to start the application containers, i.e. /run.sh (default: as defined in the image)
  • entrypoint (string) – optional, the command prefix used to start the application containers, i.e. /usr/sbin/sshd (default: as defined in the image)
  • container_ports (array(object)) – optional, an array of objects with port information to be exposed in the application containers, i.e. [{"protocol": "tcp", "inner_port": 80}] (default: as defined in the image)
  • container_envvars (array(object)) – optional, an array of objects with environment variables to be set in the application containers on launch, i.e. [{"key": "DB_PASSWORD", "value": "mypass"}] (default: as defined in the image, plus any link- or role-generated variables)
  • linked_to_application (array(object)) – optional, an array of application resource URIs to link this application to, i.e. ["/api/v1/application/80ff1635-2d56-478d-a97f-9b59c720e513/"] (default: empty array)
  • autorestart (string) – optional, whether the containers should be restarted if they stop, i.e. ALWAYS (default: OFF, possible values: OFF, ON_FAILURE, ALWAYS)
  • autoreplace (string) – optional, whether the containers should be replaced with a new one if they stop, i.e. ALWAYS (default: OFF, possible values: OFF, ON_FAILURE, ALWAYS)
  • autodestroy (string) – optional, whether the containers should be terminated if they stop, i.e. OFF (default: OFF, possible values: OFF, ON_FAILURE, ALWAYS)
  • roles (array(string)) – optional, a list of Tutum API roles to grant the application, i.e. ["global"] (default: empty array, possible values: global)
  • web_public_dns (string) – optional, a custom domain name to be used as CNAME for the application web endpoint, only available if the application listens in port 80, i.e. my-app.example.com (default: none)
Request Headers:
 
  • Content-Type – required, only application/json is supported
  • Authorization – required ApiKey authentication header in the format ApiKey username:apikey
  • Accept – required, only application/json is supported
Status Codes:
  • 202 – operation accepted
  • 400 – cannot perform the operation (probably there was a validation error on the given parameters)
  • 401 – unauthorized (wrong credentials)

Terminate an application¶

DELETE /api/v1/application/(uuid)/¶

Destroy all the containers in an application. This is not reversible. All the data stored in all the application containers will be permanently deleted.

Example request:

DELETE /api/v1/application/7eaf7fff-882c-4f3d-9a8f-a22317ac00ce/ HTTP/1.1
Host: app.tutum.co
Accept: application/json
Authorization: ApiKey username:apikey

Example response:

Images¶

List all images¶

GET /api/v1/image/¶

This operation returns a list of all jumpstarts, Linux and private images available to the user.

Example request:

GET /api/v1/image/?is_private_image=True HTTP/1.1
Host: app.tutum.co
Accept: application/json
Authorization: ApiKey username:apikey

Example response:

HTTP/1.1 200 OK
Cache-Control: must-revalidate, max-age=0
Content-Type: application/json
Vary: Accept, Authorization, Cookie

{
    "meta": {
        "limit": 25,
        "next": null,
        "offset": 0,
        "previous": null,
        "total_count": 1
    },
    "objects": [
        {
            "base_image": false,
            "cluster_aware": false,
            "description": "",
            "docker_registry": "/api/v1/registry/r.tutum.co/",
            "image_url": "",
            "imagetag_set": [
                "/api/v1/image/r.tutum.co/user/myimage/tag/latest/"
            ],
            "is_private_image": true,
            "name": "r.tutum.co/user/myimage",
            "public_url": "",
            "resource_uri": "/api/v1/image/r.tutum.co/user/myimaget/",
            "starred": false
        }
    ]
}
Request Headers:
 
  • Authorization – required ApiKey authentication header in the format ApiKey username:apikey
  • Accept – required, only application/json is supported
Query Parameters:
 
  • offset (int) – optional, start the list skipping the first offset records (default: 0)
  • limit (int) – optional, only return at most limit records (default: 25, max: 100)
  • name (string) – optional, filter applications by name
  • unique_name (string) – optional, filter applications by unique name (if name is not unique, Tutum will append a number to make it unique)
  • is_private_image (bool) – optional, display only private images
  • base_image (bool) – optional, display only Linux base images
  • starred (bool) – optional, display only jumpstart images
  • docker_registry__host (string) – optional, display only images stored in the specified host, i.e. r.tutum.co
Status Codes:
  • 200 – no error
  • 401 – unauthorized (wrong credentials)

Delete a private image from the account. Please note that this does not delete the image in the source registry.

Example request:

DELETE /api/v1/image/quay.io/user/my-private-image/ HTTP/1.1
Host: app.tutum.co
Accept: application/json
Authorization: ApiKey username:apikey

HTTP/1.1 204 No Content
Cache-Control: must-revalidate, max-age=0
Content-Type: application/json
Vary: Accept, Authorization, Cookie

  • name (string) – required, the image name to add in docker format, including the registry namespace, i.e. quay.io/user/my-private-image.

  • Authorization – required ApiKey authentication header in the format ApiKey username:apikey
  • Accept – required, only application/json is supported
